# 关于隐藏API访问警告的说明

## 警告信息

```
Accessing hidden method Landroid/view/MotionEvent;->getEventTimeNano()J (greylist, reflection, allowed)
```

## 原因

这个警告是**正常的系统警告**，来自Android系统本身，不是应用代码的问题。

### 产生原因：

1. **Material Design组件**：Material Design库内部使用反射访问一些隐藏API来实现某些功能
2. **WebView组件**：WebView在某些操作时会访问系统隐藏API
3. **Android系统机制**：这是Android的"greylist"机制，允许访问但会发出警告

### 警告级别：

- **级别**：W (Warning)
- **状态**：allowed（允许的）
- **影响**：**不影响应用功能**，只是警告信息

## 如何处理

### 方法1：忽略警告（推荐）

这是系统级别的警告，不影响功能，可以安全忽略。

### 方法2：过滤日志

如果不想看到这个警告，可以在查看日志时过滤掉：

```bash
# 只显示Translate标签的日志，过滤掉系统警告
adb logcat -s Translate:D TranslateService:D *:S

# 或者使用脚本
./view_logs.sh
```

### 方法3：在logcat中过滤

在Android Studio的Logcat中，可以添加过滤器排除这个警告。

## 总结

- ✅ 这是**正常的系统警告**
- ✅ **不影响应用功能**
- ✅ 来自Android系统或依赖库，不是应用代码问题
- ✅ 可以安全忽略

如果只想查看应用的业务日志，使用：
```bash
adb logcat -s Translate:D TranslateService:D *:S
```
